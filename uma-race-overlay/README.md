# Uma Race Overlay

Лайв-оверлей стамины (HP) и скорости всех лошадей во время гонки Umamusume,
включая лошадей других игроков в PvP (Champions Meeting / Room Match).

Работает как плагин для мода **Hachimi** (Steam-версия игры).

## Архитектура

Два компонента — намеренно разделены, чтобы рендер не мог уронить игру и не
конфликтовал с собственным GUI-оверлеем Hachimi (оба иначе цепляли бы
`IDXGISwapChain::Present`, что приводит к крашу):

1. **`uma_race_overlay.dll`** — внутриигровой плагин (грузится Hachimi).
   - Через il2cpp runtime API находит классы `Gallop.HorseRaceInfo` /
     `HorseRaceInfoReplay` / `HorseData` и ставит хуки (minhook) на:
     - `HorseRaceInfoReplay..ctor` — появление каждой лошади в гонке (имя, ворота, тренер);
     - `HorseRaceInfoReplay.get_RunMotionSpeed` — вызывается каждый кадр, отсюда читаются HP/MaxHP/скорость/дистанция/last spurt/финиш.
   - Инициализацию запускает только после загрузки `cri_ware_unity.dll`
     (= il2cpp полностью инициализирован — тот же сигнал, что использует Hachimi).
     Обращение к il2cpp раньше этого момента крашит игру.
   - Каждые 50 мс пишет снимок состояния в
     `%TEMP%\uma_race_overlay_state.json` (атомарно: temp + rename).
   - Никакой графики в процессе игры — минимальная поверхность для крашей.

2. **`uma_race_overlay_app.exe`** — отдельное приложение-оверлей (egui_overlay).
   - Прозрачное, click-through, always-on-top окно поверх игры (своё OpenGL-окно glfw).
   - Читает JSON ~20 раз/с и рисует таблицу: позиция, лошадь `[ворота] имя (тренер)`,
     бар стамины с цветом (зелёный/жёлтый/красный) и %, скорость в m/s, метку SPURT.
   - По кнопке **⚡ ускорение** (или F9) окно расширяется вправо колонками
     `accel m/s²` (текущее ускорение, зелёное при разгоне / красное при торможении)
     и `max spurt` (пик ускорения за последний спурт).
   - Таблица показывается только когда идёт гонка (`running` + свежий `ts`).
   - Хоткеи: **F8** скрыть/показать · **F9** панель ускорения · **F7** режим мыши
     (отключает click-through, чтобы кликать кнопки/галочки).

   Ускорение считает плагин: `accel = Δ_lastSpeed/Δt` со сглаживанием (EMA),
   `max_spurt_accel` — максимум `accel`, пока активен last spurt.

## Установка

1. Скопировать `uma_race_overlay.dll` в папку игры
   (`...\steamapps\common\UmamusumePrettyDerby\`).
2. В `hachimi\config.json` добавить плагин:
   ```json
   "load_libraries": ["uma_race_overlay.dll"],
   ```
3. Запустить игру через Steam.
4. Запустить `uma_race_overlay_app.exe` (можно в любой момент, до или после игры).
   - Для записи/стрима игра должна быть в **оконном или безрамочном** режиме
     (в эксклюзивном фуллскрине стороннее окно поверх D3D не отрисуется —
     в `config.json` Hachimi `auto_full_screen: false`).

## Сборка из исходников

Требуется Rust (MSVC toolchain). Для приложения-оверлея дополнительно нужен
CMake и MSVC (glfw собирается из исходников).

```powershell
# плагин (графики нет, CMake не нужен)
cd uma-race-overlay
cargo build --release

# приложение-оверлей: в окружении Developer Command Prompt (vcvars64),
# с CMake в PATH; если установлена VS новее 2022, задать генератор явно:
#   set CMAKE_GENERATOR=Visual Studio 17 2022
cd ..\uma-race-overlay-app
cargo build --release
```

## Отладка

Плагин пишет лог в `%LOCALAPPDATA%\uma_race_overlay.log`. Признак успеха:
```
race hooks installed
```
Если строки нет — смотреть, на каком шаге остановился (`resolve race api`,
имена классов/методов). Имена классов умамусуме иногда меняются с обновлениями;
тогда обновить строковые имена в `src/hooks.rs` / `src/il2cpp.rs`.

## Замечания по безопасности

Плагины Hachimi на Windows работают с полными правами процесса игры.
Этот плагин только читает память гонки и пишет локальный JSON-файл —
ничего не отправляет в сеть и не пишет в игру.
