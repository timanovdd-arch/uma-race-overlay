# Карта гонки / реплей («📈 Race map») — хэндофф

> Окно-реплей завершённой гонки: горизонтальный график (умалатор-стиль) с
> кривыми скорости/HP, маркерами скиллов на их дистанции, фазами/рельефом трассы
> и ползунком-плейхедом с живыми статами. Сделано 2026-06-17..18.

---

## 1. Что это и как пользоваться

В оверлей-приложении: хоткей **F11** или кнопка **«📈 Race map»** в нижней панели
таблицы. Окно показывает **последнюю завершённую гонку** (берётся самый свежий
файл из `%LOCALAPPDATA%\uma_race_overlay_races\`, см. [АРХИВ_ГОНОК](АРХИВ_ГОНОК.md)).
Кнопка **«⟳ Reload latest»** перечитывает свежий архив.

В окне:
- **Сверху** — девочки сеткой 3 колонки × 3 (по порядку финиша, победители первыми)
  с чекбоксами (по умолчанию показаны топ-3) и **живые статы** в текущем кадре:
  **Speed, HP, Accel (Δv/Δt), Block (да/нет)**.
- **График** (X = дистанция 0..длина трассы):
  - жирные кривые **скорости** по девочкам (свой цвет), тонкие — **HP** (тогл *Show HP*);
  - пунктир **Poskeep Gap** = отставание от лидера в метрах (тогл *Show Poskeep Gap*);
  - **полоса фаз** над треком (Opening / Middle / Last spurt) + границы по дистанции;
  - **рельеф** без заливки: границы углов тонкими линиями + серые подписи
    Corner N / Straight / ↗up / ↘down (из `course_data.json` по `course_id`);
  - **маркеры скиллов** (тогл *Show Labels*): точка на кривой скорости + выноска вниз
    к подписи в отдельной нижней зоне; подписи цветные по девочке, с тёмной подложкой.
- **Ползунок** во всю ширину графика — игрок сам тащит; двигает вертикальный
  «плейхед» и точки девочек по кривым, статы сверху обновляются вживую.

В шапке метка источника скиллов: **«● real events»** (ground truth из игры) или
**«≈ inferred from curves»** (эвристика для старых архивов, см. §3).

## 2. Где код

Приложение `uma-race-overlay-app/src/`:
- **`replay.rs`** — весь модуль реплея:
  - разбор архива (`ArchiveRace`/`ArchiveHorse`, serde) + `load_latest()`;
  - привязка скиллов `infer()` (эвристика-фолбэк) и чтение `act` (ground truth);
  - геометрия графика: 3 зоны по высоте (см. §4), упаковка подписей `build_labels`;
  - `ReplayState::ui()` (окно) + `paint_chart()` (рисование).
- **`gamedata.rs`** — добавлены `skill_names` (из `text_data`, category 47) +
  `skill_name(id)` (фолбэк `id/10` → `#id`) — для подписей скиллов.
- **`main.rs`** — поля `replay_open`/`replay`, хоткей **F11** (`poll_hotkeys`),
  кнопка в `bottom_bar`, окно `show_replay()` (геометрия курса из `self.courses`).

Плагин `uma-race-overlay/src/` (для ground truth, §3):
- **`frames.rs`** — `SkillEvent` + `collect_skill_events(sim, nh)` + поле
  `Timeline.events`.
- **`archive.rs`** — пишет на лошадь массив `"act":[{f,d,id}]`.
- **`hooks.rs`** — одноразовый дамп `_simEvDataList` (диагностика формата).

## 3. Тайминг скиллов — ДВА источника

В архиве лежат **надетые** скиллы, но не «когда сработал». Решено двумя путями:

### A. Эвристика (фолбэк, работает на любых архивах)
По реальным кривым `v`/`hp` детектируем события: скачок HP вверх → recovery;
рывок ускорения над локальной медианой → speed-скилл. Привязка к надетому скиллу
по типу эффекта (master.mdb) + условиям-DSL (`phase`/`remain_distance`/`hp_per`).
Приблизительно (метка «≈ inferred»), `confidence` управляет прозрачностью маркера.

### B. Ground truth (точно, из самой игры) — ОСНОВНОЙ
Гонка предрассчитана; события лежат в `RaceSimulateData._simEvDataList` (+0x28,
`List<T>`, backing `_items`+0x10). **Раскладка элемента
`RaceSimulateEventData`** (ref-тип, stride 8), РАСКОДИРОВАНА:

| смещение | поле | тип |
|----------|------|-----|
| +0x10 | `frameTime` | f32 (сек) |
| +0x14 | `type` | i32 |
| +0x18 | `param` | `int[]` (ref); элементы i32 с +0x20 |

**`type == 3` = активация скилла**, `param = [horseIdx, skillId, duration, 0, 1<<horseIdx]`,
где `duration` в 1e-4 c (0 = мгновенный). События `t=0` с `param[2] == -1` — это
**пред-стартовая регистрация** надетых скиллов (НЕ активация), отбрасываются по
`param[2] < 0`. Типы 4/5 (param длиной 1, один horseIdx) — иные события (поз-кип/
спурт?), для скиллов не используются.

`horseIdx` = тот же индекс кадрового массива, что и таймлайн (архив привязывает как
`gate-1`), поэтому маркеры ложатся ровно на нужную кривую. Плагин на финише пишет
эти события в архив (`act`), окно предпочитает их и показывает «● real events».

> Проверено на реальной гонке: у Seiun Sky события совпали 1:1 с надетыми скиллами
> и порядком в игре (ульта «Angling and Scheming» @1470м → Front Runner Corners →
> Barcarole). Все `act`-id входят в её надетый набор — маппинг верный.

## 4. Геометрия графика (важно для правок UI)

График **КОНТЕНТ-РАЗМЕРНЫЙ** по высоте — 3 зоны (константы в `replay.rs`):
`PHASE_H`(26) полоса фаз сверху · `CURVE_H`(190) зона кривых · зона подписей
высотой `n_rows × ROW_H`(17) · `AXIS_H`(18) ось дистанции снизу. `n_rows` =
число строк из `build_labels` (greedy-упаковка подписей по дистанции), поэтому
**скиллы трёх лошадей никогда не накладываются**, а высота не «прыгает». Ширина:
`available_width().clamp(720, 1200)`.

**Грабли egui (записаны и в памяти):**
- `ui.separator()` ВНУТРИ `horizontal/horizontal_top` = ВЕРТИКАЛЬНЫЙ сепаратор,
  тянется на всю available height; если в строке нет высокого соседа, он раздувает
  строку на всю высоту окна → гигантский зазор. **Между колонками — `ui.add_space`,
  НЕ separator.**
- Не брать `available_height()` для высоты painter (окно «прыгает»/тянется).
- Не брать `available_width()` без clamp (растягивает окно на все мониторы).
- Окно egui помнит размер в рамках сессии → правки размеров видны после
  ПЕРЕЗАПУСКА приложения.

## 5. Полоса «Field» + Spot Struggle / Dueling (2026-06-18)

### Полоса «Field» (нижняя зона, тогл «Show Field»)
Под осью дистанции — зона `FIELD_H`(88)+`FIELD_GAP`(8). X = дистанция (та же шкала),
Y = `LanePosition` (lane; авто-шкала min/max показанных, меньший lane = внутренняя
бровка внизу). Для каждой показанной лошади: базовая кривая lane её цветом + оверлеи
состояний поверх: **красный = блок** (`blocker_at`), **оранжевый = Spot Struggle**
(`struggle_partners`), **фиолетовый = Dueling** (`duel_partners`) + плейхед-точки.
Наглядно видно, где была давка/блок и где контест.

### Live stats: колонки Block / Struggle / Duel
- **Block** — «yes (by X)»: `BlockFrontHorseIndex` = индекс лошади спереди (ground
  truth). Имя из `race.horses[idx]` (массив по gate = тот же индекс).
- **Struggle** — «yes (with X)»: Spot Struggle.
- **Duel** — «yes (with X)»: Dueling (только ground truth, пусто у старых архивов).

### ГЛАВНОЕ: где спот-страгл/дуэль в движке (реверс 2026-06-18)
- **НЕ per-frame поле.** Дамп класса `RaceSimulateHorseFrameData`
  (`%LOCALAPPDATA%\uma_race_overlay.log`, «fields of HorseFrame») перечисляет ВСЕ
  поля кадра: только `Distance/LanePosition/Speed/Hp/TemptationMode/BlockFrontHorseIndex`.
  Block лежит покадрово (пространственная связь каждый кадр), контест — нет.
- **Контесты — события `_simEvDataList`:** `type==4` = **Spot Struggle (位置取り争い)**,
  `type==5` = **Dueling (追い比べ)**; `param[0]` = индекс лошади-участника; на каждого
  участника отдельное событие в один `frameTime` → партнёров группируем по кадру.
- **Подтверждено** гонкой `race_1781730299463`: type 4 @t=7.86s для Seiun Sky + Silence
  Suzuka (оба nige, 2.6 м, ~140 м); type 5 на финишной прямой (t≈94 с). Редкость
  событий (в логе 4×type4, 6×type5) = «происходит далеко не всегда».
- Правило-фолбэк для спот-страгла (`struggle_partners_rule`, два передних nige в 3.75 м,
  окно 150 м..5/12) ПЕРЕ-СРАБАТЫВАЕТ (≈21/22 гонок) — движок селективнее. Поэтому
  ground truth из событий главнее; правило — только для старых архивов («≈ derived»).

### Пайплайн ground truth
- Плагин `frames.rs::collect_contest_events`(type 4/5) → `Timeline.contests`;
  `archive.rs` пишет на лошадь `"struggle":[{f,d}]` и `"duel":[{f,d}]`.
- App `replay.rs`: `ArchiveHorse.struggle/duel` → `build_contest_eps` (партнёры по
  кадру старта ±1, конец пока в пределах `CONTEST_SEP_DIST`=4.5 м от партнёра) →
  `struggle_partners`/`duel_partners`. Шапка: «⚔ real struggle/duel» показывается
  ТОЛЬКО когда события реальные. Метку «≈ derived struggle» убрали из шапки
  (2026-06-18, рестайл UI) — правило-фолбэк всё равно работает в таблице/поле.
- Харнесс: `scan_all_archives_for_struggle` (`#[ignore]`) — сканит ВСЕ архивы правилом.
- ⚠ Старые 22 архива записаны до этого плагина → событий НЕТ → работает правило-фолбэк
  (без бейджа). Нужно: пересобрать DLL, скопировать в игру при ЗАКРЫТОЙ игре, сыграть 1 гонку.

## 5. Сборка / запуск

ТОЛЬКО `--release` (debug glfw падает без cmake; release-glfw закэширован):
```sh
cargo build --release --manifest-path uma-race-overlay/Cargo.toml      # плагин (.dll)
cargo build --release --manifest-path uma-race-overlay-app/Cargo.toml  # приложение (.exe)
cargo test  --release --manifest-path uma-race-overlay-app/Cargo.toml  # тесты replay.rs
```
`course_data.json` должен лежать рядом с exe (раскраска рельефа). DLL — в папку
игры; новые `act`-данные появляются у гонок, сыгранных уже с обновлённым плагином.
