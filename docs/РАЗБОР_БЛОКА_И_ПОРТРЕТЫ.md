## 1. UI-правки оверлея (готово, протестировано)

- Колонка ускорения разбита на два независимых тоггла:
  - **«base accel live»** (F9) → колонка `accel m/s²` (текущее живое ускорение).
  - **«spurt phase data»** → `max spurt accel` + `max spurt speed`.
- Плагин считает новое поле `max_spurt_speed` (пик скорости в спурте, обновляется
  покадрово при `is_last_spurt` в `run_motion_speed_hook`).
- **Фикс «ников квадратами»** (CJK/кириллица): `install_system_fonts()` в `main.rs`
  подмешивает системные шрифты Windows (YuGothic/MS Gothic/Malgun/YaHei/SimSun из
  `C:\Windows\Fonts`) как фолбэк egui. Не бандлим в exe. Подтверждено живым логом —
  чужие тренеры бывают на не-латинице (напр. `凛雨翔`).
- Чистка таблицы: убран пустой столбец-индикатор SPURT и вертикальные разделители;
  индикатор спурта встроен прямо в ячейку **Speed** (оранжевый `⏫`, hover «SPURT»).

## 2. Блок-анализ реальной гонки (готово)

Цель: по ДАННЫМ ИГРЫ (без симуляции) показать, сколько лошадь была в блоке и
повлиял ли блок на её место.

### Реверс (оффсеты подтверждены живой игрой)
- Гонка **предрассчитана** при загрузке (`ImportBase64`) → все покадровые данные
  всей гонки уже в `_simData` на старте.
- `RaceSimulateReader` +0x10 `_simData`.
- `RaceSimulateData`: +0x18 `_frameDataList`(List<FrameData>), +0x20
  `_horseResultDataArray`. List backing array `_items` в +0x10.
- `RaceSimulateFrameData` (class): +0x10 `Time`(f32), +0x18 `HorseDataArray`.
- `RaceSimulateHorseFrameData`: +0x10 `Distance` +0x14 `LanePosition` +0x18 `Speed`
  +0x1c `Hp` +0x20 `TemptationMode`(u8, закидывание/掛かり)
  **+0x21 `BlockFrontHorseIndex`(u8; 0xFF = свободна, иначе индекс блокирующего)**.
  Может быть value-типом → читать с поправкой `adj=0x10`.
- `RaceSimulateHorseResultData`: +0x10 `FinishOrder`(0-based) +0x14 `FinishTime`
  +0x18 `FinishTimeRaw` +0x1c `FinishDiffTime`(отрыв до победителя, сек)
  +0x28 `LastSpurtStartDistance` +0x2c `RunningStyle` +0x30 `Defeat`.
- **Маппинг idx кадрового массива = `gate-1`** (подтверждено: в логе всюду
  `simFO == liveFO-1`, наш liveFO = `get_FinishOrder+1`).

### Хелперы il2cpp (добавлены)
`class_get_element_class`, `is_valuetype`, `array_element_size`
(`il2cpp_class_array_element_size`), `array_elem_base` (инлайн-данные для value-типа
или разыменование для ссылочного). Берут тип/страйд из КЛАССА массива — без
разыменования возможных struct-элементов (без риска краша).

### Модуль `frames.rs` (новый)
`compute(reader)` — на финише гонки (все `finished`, one-shot `STATS_DONE`) один
проход по кадрам → per-horse: `blocked_time`/episodes (полный блок, для справки),
`kakari_time`, и **отдельно блок В СПУРТЕ** (`Distance >= LastSpurtStartDistance`,
фолбэк — последняя 1/6 трассы). `RACE_READER` стэшится в ctor, сбрасывается на
новой гонке вместе с `STATS_DONE`.

### Ключевой урок
«Формально в блоке» ≠ «реально тормозит»: gate8 был 19с в блоке, а потеря лишь
0.15с (ехал на той же скорости за спиной). Поэтому метрика потери = **дефицит
скорости**, а не время в блоке. И блок ВНЕ спурта почти не вредит — считаем
именно спурт-блок (он глушит ускорение на решающей стадии).

### Оценка «места без блока» (counterfactual)
- Потеря в спурте оценивается по **скорости освобождения**: эпизод буферизуется
  (вход/длительность/пройдено), на снятии блока опорная скорость = `max(вход,
  скорость сразу после освобождения)` — лошадь там рвёт вперёд, это и есть
  «задушенное ускорение». `lost_dist = max(0, refv*dur − dist)`, `lost_time = …/refv`.
- Зажат до финиша (эпизод не закрылся) → опора = входная скорость (занижено),
  флаг `spurt_unresolved` → в UI помечается `?` (свободную скорость из одной гонки
  узнать нельзя; точный ответ — только стат-калькулятор).
- **UI колонка «place w/o block»**: контрфактуальное место, если вернуть спурт-
  потерю ВСЕМ и пересортировать финишные времена
  (`ct = finish_diff_time − spurt_lost_time`, сортировка по возрастанию). Показывает
  `P5→P2` (был 5-м, без блоков — 2-й). cf==1 красным, иначе оранжевым; без
  изменения — серое `P{rank}`. Считается по всему составу ДО фильтрации (как rank).
- Колонка «spurt block» = время×эпизоды блока В СПУРТЕ; полный блок/закидывание — в тултипе.

### Эволюция метрики (история, чтобы не повторять)
1. Сначала «race kill» показывал отрыв до соперника впереди → юзер: «маловато»
   (это финишные десятые секунды плотной гонки, а не потеря).
2. Затем «win w/o block?» YES/no по `lost ≥ gap_to_winner` → наивно (только я vs
   победитель; lost занижен на флэт-блоке).
3. Итог: контрфактуал по всему полю + опора по освобождению + флаг `?`.

## 3. Портреты персонажей (решено, не реализовано)

- ID-геттеры HorseData есть: `get_CardId` (карта=персонаж+костюм, ключ иконки),
  `get_RaceDressId`, `get_TrainedCharaId`, `get_IsMob`/`get_MobId` (мобы — без портрета).
- **Решение: источник = игра (вариант B), доставка = локальный пак**
  `portraits/{card_id}.png` рядом с exe; рендер = тот же PNG→egui-текстура, что у QR.
- **НЕ рантайм-извлечение:** `…\LocalLow\Cygames\umamusume\meta` ЗАШИФРОВАН (не
  «SQLite format 3», начинается с `\xc6\x0e\xf3\x9d`); `dat/{xx}/{hash}` — типы
  вперемешку (видел `AFS2` = CRIWARE-аудио; иконки — отдельные UnityFS). Расшифровка
  meta + парсинг UnityFS + декод Texture2D в рантайме Rust = хрупкая большая
  подсистема. Делать офлайн разово (UnityPy + известный ключ глобалки) → PNG-пак.
- **ОСТАЛОСЬ:** чтение card_id/is_mob (плагин→JSON) + рендер иконки в колонке Horse
  с кэшем и фолбэком на текст; отдельный скрипт-выгрузка иконок из игры.

## 4. Сборка / деплой (как делали)

- Плагин: `cargo build --release --manifest-path uma-race-overlay/Cargo.toml`.
- App: `cargo build --release --manifest-path uma-race-overlay-app/Cargo.toml`
  (перед сборкой убить exe: `Get-Process uma_race_overlay_app | Stop-Process -Force`).
- Деплой dll: только при ЗАКРЫТОЙ игре →
  `C:\Program Files (x86)\Steam\steamapps\common\UmamusumePrettyDerby\uma_race_overlay.dll`
  и в `UmaRaceOverlay-RELEASE\`. Exe — в `UmaRaceOverlay-RELEASE\` (можно при открытой игре).
- Лог плагина: `%LOCALAPPDATA%\uma_race_overlay.log`; строки `frame stats:` —
  валидация маппинга и значения блока.
