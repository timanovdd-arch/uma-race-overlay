# Авто-детект курса трассы (course_id / track_id) — точка входа нового чата

> Это промпт для нового чата + дев-контекст. Цель: чтобы плагин САМ определял
> курс текущей гонки и слал его в JSON, а app подхватывал реальную геометрию
> автоматически (сейчас курс выбирается вручную в UI).

---

## ЗАЧЕМ

Винрейт-симулятор стал track-aware: реальная геометрия трассы (углы/уклоны/прямые из
`course_data.json`, 138 курсов) определяет, ГДЕ срабатывают accel-скиллы — а это решает
гонку (accel на входе в спурт = отрыв). Геометрия выбирается по `course_id`. Сейчас юзер
ставит `course_id` ВРУЧНУЮ (cfg/UI). Нужно — авто.

## ЧТО УЖЕ ТОЧНО ИЗВЕСТНО (не повторять зря)

- **course/track id НЕТ ни в одном объекте, который плагин хукает.** Продамплены (лог
  `%LOCALAPPDATA%\uma_race_overlay.log`, дампер в `hooks.rs` под флагом SKILL_DUMPED):
  - `HorseData` (полный список методов) — есть get_GateNo/Raw*/ActiveProperDistance/
    WinRaceInstanceIdList и пр., но **геттера курса/трассы НЕТ**.
  - `HorseRaceInfoReplay` (хукаемый `this`) — поля +0x260 `_simulateReader`, rotation,
    temptation; метод `GetBehindHorseNearLaneContinueTime(1)` (полезно для позиционки!),
    но **курса НЕТ**.
  - `RaceSimulateReader` — поля _simData/_curTime/_eventDataAsc/TopFinish*/_importer; методы
    GetDistance(2)/GetHorseResultData/get_SimulateData/ImportBase64 — **курса НЕТ**.
  - `RaceSimulateData` — _frameDataList/_horseResultDataArray/_simEvDataList/_horseNum/
    _distanceDiffMax/_lastFrameIdx… + `<Header>k__BackingField`(=`RaceSimulateDataHeader`,
    это ПРОСТО версия сериализации, не метаданные). int-скан этих объектов на значения
    10xxx/дистанцию — **пусто**.
- **Вывод:** course/track id живёт в объекте УРОВНЯ ГОНКИ (RaceManager / RaceInformation /
  RaceData / RaceDefine), который плагин пока НЕ трогает.
- **Матч без точного id не годится:** distance+surface даёт 6 курсов на 2200turf;
  distance+surface+**track_id** уникален в 130/134. Значит нужен либо `course_id` (=
  `race_course_set.id`, ключ course_data, напр. Hanshin 2200 = 10906), либо `track_id`
  (=raceTrackId, напр. 10009) + distance + inout.

## ПЛАН (новый чат)

1. **Найти источник курса.** Кандидаты по убыванию вероятности:
   - `Gallop.RaceManager` (или `RaceManager`) — синглтон со static `Instance`; навигировать
     к RaceInformation/RaceData → `raceCourseSetId`/`courseId`/`raceTrackId`/`distance`.
   - `RaceScenario` / `RaceData` (то, что парсит hakuraku из `race_scenario` пакета) — курс
     там есть; найти, где игра держит распарсенный сценарий.
   - Способ: `il2cpp::find_class(image,"Gallop","RaceManager")` → `dump_class_fields/methods`
     → найти static instance (il2cpp_field_get_offset для static / il2cpp_field_static_get_value)
     → следовать полям. Хелперы уже есть в `il2cpp.rs` (возможно добавить чтение static-полей).
2. **Прочитать course_id** в `ctor_hook` (reader/this уже там) или в новом хуке → сверить с
   ключами `course_data.json` (= master.mdb `race_course_set.id`).
3. **Слать в JSON** верхним полем `course_id` (или `track_id`+distance) — в `state.rs`/`writer.rs`.
4. **App:** в `apply_sim` брать `course_id` из снапшота (вместо/поверх ручного `cfg.course_id`).
   Поле и UI уже готовы — заменить источник.

## ТЕХ-КОНТЕКСТ (где что в коде)

- Плагин `uma-race-overlay/`: `il2cpp.rs` (биндинги+хелперы: find_class/method, field_offset,
  dump_class_fields/methods, read_field, class_of/name), `hooks.rs` (ctor_hook(this=
  HorseRaceInfoReplay, data=HorseData, reader=RaceSimulateReader); блок SKILL_DUMPED —
  туда добавлять дампы; RACE_READER стэшится), `writer.rs`/`state.rs` (JSON).
- Сборка dll: `cd uma-race-overlay && cargo build --release` → `target/release/uma_race_overlay.dll`.
- Тест-цикл: закрыть игру → подменить dll в папке игры → зайти в гонку → читать лог
  `%LOCALAPPDATA%\uma_race_overlay.log` (дамп one-shot per launch).
- App `uma-race-overlay-app/`: `course.rs` (CourseGeom + load_courses), `apply_sim` в main.rs
  (course_id→geom→RaceParams.course), `Cfg.course_id`+UI «course:» в bottom_bar. Таблица
  грузится из `course_data.json` рядом с exe.
- ⚠ `course_data.json` — GPLv3 (uma-skill-tools), dev-only, в .gitignore. Для релиза —
  сгенерировать свою копию геометрии из игры.
